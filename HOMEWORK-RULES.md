# Правила сдачи домашних заданий на курсе

Все домашние задания на курсе сдаются через github. Вы делаете приватный репозиторий, в котором настраиваете upstream и сдаете задания через pull request. Далее приведена подробная инструкция. Если с каким-либо из шагов у вас возникнут проблемы, не стесняйтесь задавать их в чате курса. 

**Пошаговая инструкция (как настроить репозиторий):**

1. Создайте пустой репозиторий. Название может быть любым (но лучше, чтобы оно было связано с названием курса). **ВАЖНО!:** репозиторий обязательно должен быть приватным.
2. Вам необходимо добавить преподавательский состав (преподавателей и ассистентов) в свой репозиторий. (`settings -> collaborators -> add people`). Ники находятся на главной странице [README.md](./README.md). Для удобства, мы продублируем их в канале. 
3. Клонируете репозиторий на свой компьютер. (`git clone <АДРЕС РЕПОЗИТОРИЯ>`)
4. Настраиваете upstream с этим репозиторием – `git remote add upstream https://github.com/Inkln/CV-HSE-MIPT-Course-2026-spring.git`
5. Делаете pull upstream репозитория – `git pull origin upstream`
6. На данном этапе у вас должна получиться точная копия данного репозитория. Если это не так, можете проконсультироваться в чате курса, мы вам поможем.
7. Далее вы будете сдавать домашние задания, через новую ветку и создание pull request. Все шаги описаны в следующей инструкции

**Пошаговая инструкция (как сдавать домашние задания):**

1. Прежде всего переходим в `main` ветку – `git checkout main` 
2. Перед тем, как приступить к выполению, убедитесь, что вы обновили свой репозиторий с помощью команды `git pull upstream main`.
3. Создаем новую ветку для выполнения домашнего задания. Название ветки задаем в формате `lesson_<НОМЕР>_hw`. Например, для сдачи 1го задания нужно будет создать ветку `lesson_1_hw` – `git checkout -b lesson_1_hw`
4. Усердно трудимся, чтобы выполнить все задания
5. Добавляем и пушим файлы `git add <ФАЙЛЫ> && git commit -m '<КОММЕНТАРИЙ>' && git push origin lesson_<НОМЕР>_hw` (в первый пуш может потребоваться `git push --set-upstream origin lesson_<НОМЕР>_hw`)
6. Создаем pull request. Для этого переходим на страницу репозитория, выбираем созданную ветку и нажимаем кнопку `Compare & pull request`. В заголовке pull request указываем название задания.
7. Ждем, пока преподаватели проверит и прокомментируют ваше решение.
8. **ВАЖНО!** не делаем коммиты в ветку после дедлайна, до комментариев от преподавателей. Проверяющий ваше задание будет ориентироваться на дату сдачи задания по дате последнего коммита. Если вы будете коммитить после дедлайна, то можете случайно получить снижение баллов
9. Если проверяющий оставил вам комментарии с просьбой доработки – вы можете это поправить, и это не будет считаться сдачей после дедлайна. Пример: `Привет! Классно справился с задачей, однако код не отфарматирован и плохо читаем. Пока что ставлю 9, если поправишь, то поставлю 10.`
10. Если вы делаете доработки по комментариям проверяющего, пожалуйста, тегните его, чтобы проверить ваши изменения (делается в комментарии к pull request). 
11. **ВАЖНО!** pull request не надо сливать в `main`, чтобы избежать проблем в последующем (если смержите, то `git pull upstream main` может не пройти).
